===========
Class types
===========

-------
Configs
-------
Classes containing mission configuration settings. Should be non-static for testing purposes.
Always ends with "Config".

-------
Handlers
--------
Global classes that performs global actions regarding the area defined by its name.
Always ends with "Handler".

-------
Objects
-------
Non static classes modeling general objects that can be used as parameters and sent over the network.

-------
Services
--------
Static classes with a public StartAsync method and a private Run method, performing its own task concurrently.
Always ends with "Service".
